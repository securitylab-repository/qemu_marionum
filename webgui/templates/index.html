Aide — EFREI PARIS QEMU WIFI LAB

                
            

            


                
Présentation

                
Interface web pour déployer des labs multi-VMs basés sur QEMU + VDE (Virtual Distributed Ethernet).
                Les VMs communiquent entre elles via un switch virtuel userspace, sans accès root sur le host.

                
L'interface utilise une architecture per-VM : chaque VM choisit indépendamment son backend, son image disque, sa RAM, etc.
                Le canvas central affiche la topologie en temps réel.


                
Démarrage rapide

                
    
                    
  1. Sélectionnez une image disque globale — champ obligatoire, utilisé par défaut pour les VMs sans disque spécifique (bouton "Parcourir...")
    2. 
                    
  2. Ajoutez des VMs — glissez l'icône "+ VM" depuis la palette sur le canvas (le canvas est vide au démarrage)
    3. 
                    
  3. Configurez chaque VM — cliquez sur une VM pour ouvrir son panneau : backend, disque, RAM, CPU, label, paquets
    4. 
                    
  4. Ajoutez un serveur vwifi (optionnel) — glissez l'icône "+ Serveur vwifi" si vous utilisez un backend vwifi
    5. 
                    
  5. Cliquez "Lancer" — démarre le switch VDE + toutes les VMs. Chaque VM s'ouvre dans un xterm
    6. 
                    
  6. Cliquez "Stopper" — arrête les processus (l'état est préservé dans /tmp/vde/)
    7. 
                


                
Interface

                

                    
ZoneDescription

                    
Palette (gauche)Icônes drag-and-drop : "+ VM" et "+ Serveur vwifi". Bouton "- VM" pour retirer la dernière VM

                    
Canvas (centre)Topologie SVG : VMs, switch VDE, NAT, serveur vwifi. Drag pour déplacer les noeuds

                    
Configuration (droite)Options globales (disque, RAM, réseau) et panneau per-VM (apparaît au clic sur une VM)

                    
Sortie (bas)Logs du lab, commandes Wireshark. Redimensionnable par le handle du haut

                

                
La palette et le panneau de configuration sont redimensionnables en tirant les séparateurs verticaux entre les panneaux.


                
Cycle de vie du lab

                

                    
BoutonActionÉtat après

                    
LancerDémarre un nouveau lab (nettoie l'ancien si existant)Running

                    
StopperTue les processus, préserve disques/scripts dans /tmp/vde/Stopped

                    
RedémarrerRecrée le switch VDE et relance les VMs depuis les scripts préservésRunning

                    
NettoyerTue les processus et supprime /tmp/vde/ (retour à zéro)Idle

                

                
En mode overlay/copy, le redémarrage réutilise les disques existants (les données sont conservées).

                
Note : supprimer une VM ou le serveur depuis le canvas nettoie automatiquement /tmp/vde/ pour éviter les conflits avec les anciens scripts.


                
Configuration per-VM

                
Chaque VM possède sa propre configuration, indépendante des autres :

                

                    
ChampDescription

                    
LabelNom affiché sur le canvas (ex: "Attaquant", "Serveur web")

                    
BackendMode de configuration : fw_cfg, cloud-init, vwifi (Debian), vwifi (Alpine)

                    
DisqueImage disque spécifique (si vide, utilise le disque global)

                    
RAM / CPURessources dédiées (si vide, utilise les valeurs globales)

                    
Mode disquesnapshot, overlay, copy ou shared (si vide, utilise le mode global)

                    
PaquetsFichier liste de paquets à installer au boot

                

                
Cliquez sur une VM dans le canvas pour afficher son panneau. Le bouton "Config globale" revient aux options partagées.


                
Les 4 backends

                

                    
BackendOSConfigurationUsage

                    

                        fw_cfg
                        Alpine Linux
                        Scripts injectés via QEMU fw_cfg
                        VMs légères, boot rapide

                        Alpine Linux
                        Comme fw_cfg + simulation WiFi (mac80211_hwsim)

Chaque VM choisit son propre backend. Les backends vwifi nécessitent un serveur vwifi, à ajouter depuis la palette (icône "+ Serveur vwifi").


                
Serveur vwifi

                
Le serveur vwifi est un noeud dédié qui coordonne la simulation WiFi entre les VMs. Il doit être ajouté explicitement depuis la palette.

                
    
                    
  • Configurable : backend (fw_cfg / cloud-init), RAM, CPU, disque, mode disque
    • 
                    
  • Adresse IP fixe : x.x.x.2 sur le réseau VDE
    • 
                    
  • Supprimable via le bouton × sur le canvas
    • 
                    
  • Déplaçable par drag sur le canvas
    • 
                


                
Modes disque

                

                    
ModeDescription

                    
snapshotVolatile — les modifications sont perdues à l'arrêt (RAM uniquement)

                    
overlayDelta qcow2 — les modifications sont sauvegardées dans un fichier séparé

                    
copyCopie complète du disque — indépendant de l'original

                    
sharedÉcrit directement sur l'image — dangereux, utiliser avec précaution

                


                
Gestion individuelle des VMs

                
    
                    
  • Clic gauche sur une VM → ouvre le panneau de configuration per-VM
    • 
                    
  • Clic droit sur une VM ou le serveur → menu contextuel : Démarrer / Arrêter / Configurer
    • 
                    
  • Drag sur une VM, le switch ou le serveur → déplacer sur le canvas
    • 
                    
  • Bouton × sur une VM ou le serveur → supprimer du canvas
    • 
                    
  • Les VMs peuvent être arrêtées et redémarrées individuellement une fois le lab lancé
    • 
                    
  • Fermer le xterm d'une VM = arrêter la VM (SIGHUP tue le processus QEMU)
    • 
                


                
Port mirroring — Wireshark

                
Permet de capturer tout le trafic inter-VMs dans Wireshark sans mode HUB global.

                
    
                    
  1. Lancez le lab normalement
    2. 
                    
  2. Cliquez le bouton "Wireshark" (violet) dans le panneau de sortie
    3. 
                    
  3. L'application crée un pipe nommé et lance vdecapture
    4. 
                    
  4. Exécutez la commande affichée dans un terminal :
    
                        sudo wireshark -k -i /tmp/vde/vde.pipe
    5. 
                    
  5. Une fois Wireshark connecté, le port est automatiquement passé en mode HUB
    6. 
                    
  6. Pour arrêter : cliquez "Stop mirror"
    7. 
                

                
Prérequis : vdecapture (compiler depuis
                github.com/virtualsquare/vdecapture),
                socat, et wireshark.


                
Réseau

                
VM1 (192.168.100.10)    VM2 (192.168.100.11)    VMn
       |                       |                  |
       +----------- VDE switch (/tmp/vde/switch) -+
                               |
VM1 (10.0.2.15)         VM2 (10.0.2.15)
       |                       |
       +---- NAT QEMU (10.0.2.2) ---> Internet

                
    
                    
  • Réseau VDE : switch virtuel inter-VMs (configurable : CIDR, IP de base)
    • 
                    
  • NAT QEMU : accès Internet + redirection SSH par port (désactivable avec "Pas de NAT")
    • 
                    
  • Accès SSH : ssh debian@localhost -p 2222 (port = base SSH + index VM)
    •
